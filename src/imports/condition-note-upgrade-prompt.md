[FIGMA MAKE UPGRADE PROMPT — 컨디션노트 v2 (40+ Role-Based 운영 강화)]

역할: 당신은 “컨디션노트(Condition Note)” 헬스/치유캠프 플랫폼의 시니어 UI/UX 디자이너이자 제품 설계자입니다.

현재 상태:
- 참가자(40대 이상 성인)용 모바일 화면은 이미 충분히 만들어져 있음(유지해야 함).
- 하지만 보호자(참가자의 자녀), 관리자(치유마을 운영자), 코디네이터(프로그램 진행자) 화면이 빈약함.
- 목표를 “40대 이상 건강관리 + 생체리듬(수면/일주기) 최적화 지원 + 인지 건강(치매 예방 지원) + 정서 안정/우울 예방 지원” 중심으로 업그레이드해야 함.
- 기존 디자인 톤/컴포넌트 스타일을 최대한 유지하면서 확장(완전 재설계 금지). 기존 UI에서 색/카드/타이포/간격을 그대로 계승한 “확장형”으로 작업.

필수 전제:
- iOS / Android 완전 동일 UI(100% 동일 레이아웃).
- 출력 텍스트/카피/라벨은 모두 한국어.
- 의료 진단/확정/치료를 암시하지 말 것. “지원/관리/권고” 표현으로 설계.
- 측정기/워치 데이터는 ‘직접 연동 UI’가 아니라: 측정 → 서버 저장 → 앱 조회 방식.
- 이름 + 생년월일 일치 시 자동 데이터 다운로드(기존 흐름 유지).
- 분석 항목: 자율신경능력/수면패턴/스트레스/혈관단계/피로도 + (추가) 생체리듬 안정도/인지 루틴/정서(기분) 루틴.
- AI 기능은 RAG(FAISS) + ChatGPT 3.5 Turbo 기반 요약/권고를 전제로 “UI상 표현”만 설계(모델 구현 논의 X).

작업 목표:
1) 보호자(자녀) 앱/페이지 강화: “안심 + 변화 + 액션 + 동의/권한”
2) 관리자(치유마을) 운영 콘솔 강화: “식단 메뉴 + 치유프로그램 + 운영 대시보드 + 리포트”
3) 코디네이터 콘솔 강화: “오늘 할 일 + 현장 체크인 + 케이스 기록 + 에스컬레이션”

1) 보호자(자녀) 앱 — “가족 케어 대시보드”
------------------------------------------------
목적: 안심 + 변화 이해 + 필요 시 액션(연락/예약/문의)

필수 화면 8개(모바일 우선, 필요 시 태블릿 변형):
G1. 보호자 홈(요약 대시보드)
- 상단: 참가자 프로필(이름/나이대/캠프 기간) + 공유 범위 배지
- 핵심 카드 3개:
  1) 오늘 상태(정상/주의/관리필요) + 한줄 요약
  2) 최근 7일 변화(좋아짐/유지/악화) + 핵심 지표 2~3개 스파크라인
  3) 권장 액션(예: 통화 권장/코디네이터 상담 권장/휴식 권장)
- 알림 센터(위험/주의/정보 탭) + “확인 완료” 처리
- 하단 CTA: “주간 리포트 보기”, “코디네이터 문의”

G2. 변화 추이(7/30/90일)
- 수면/스트레스/생체리듬 안정도/피로도 추이를 탭으로 전환
- AI 해석 1~2문장(과장 금지)
- “이번 주 특징” 카드(규칙성/분절/회복)
- 비교선(개인 평균선) 정도만 제공(연령 평균 비교는 옵션)

G3. 위험 신호 상세
- 위험 트리거 로그(언제부터/무엇이/권고)
- “권장 조치” 카드(코디네이터 상담, 생활 루틴 점검)
- 보호자 액션 버튼:
  - 통화/메시지 템플릿
  - 코디네이터에게 전달(메모 첨부)

G4. 주간/월간 리포트
- 핵심 지표 요약 + 변화
- 프로그램 참여 요약(참가자 동의 범위 내에서만)
- 다음 주 권장(수면/활동/사회활동/인지 루틴)
- 공유 버튼(가족 공유/저장/PDF)

G5. 프로그램 참여 현황(공유 허용 범위 내)
- 오늘/이번 주 참가한 프로그램
- 참여율, 만족도(있다면)
- “참여를 독려하는 메시지” 예시(정중한 톤)

G6. 코디네이터 문의/예약
- 문의 채널(메시지/통화 예약)
- 예약 캘린더(시간 선택 UI)
- 사전 질문 템플릿(수면/식사/기분/활동)

G7. 공유 동의 상태 보기(읽기 전용)
- 참가자가 설정한 공유 범위 표시
- 변경은 참가자 앱에서만 가능(안내 문구)

G8. 보호자 설정
- 알림 수신 조건(위험만/주의 포함/전체)
- 리포트 수신 주기(주 1회)
- 개인정보/동의 관련 안내

상태 설계:
- 데이터 없음(아직 공유 전/측정 전)
- 공유 제한 상태(일부 가림)
- 측정 누락(“최근 측정이 없습니다”)
- 위험 알림 폭주 방지(묶음 알림 UI)

2) 코디네이터 콘솔 — “현장 운영 + 케이스 관리”
------------------------------------------------
목적: 오늘 운영/체크인/기록/에스컬레이션

필수 화면 10개:
C1. 코디네이터 홈(오늘 운영 허브)
- 오늘 일정 타임라인(세션 카드)
- 우선 확인 대상(관리필요/주의 리스트)
- 측정 누락/불참자 리스트
- 보호자 문의 큐
- 빠른 액션: “체크인 시작”, “기록 작성”, “관리자에게 보고”

C2. 참가자 리스트(필터/검색)
- 위험도 배지(정상/주의/관리필요)
- 최근 24시간 변화 아이콘
- 필터: 위험도/측정누락/프로그램 불참/수면악화/스트레스급등
- 정렬: 위험 우선, 최근 이벤트 우선

C3. 참가자 상세(케이스 파일)
- 상단: 프로필 + 공유 동의 배지 + 담당 코디네이터
- 핵심 지표 카드(수면/스트레스/회복/리듬/피로)
- 14일 추이 + 리듬(취침/기상 규칙성) 시각화
- 오늘 권장 프로그램/주의사항
- “현장 메모” 영역(태그: 수면/기분/통증/컨디션 등)
- 버튼: “조치 계획 추가”, “관리자/보호자(동의시)에게 공유”, “이벤트 기록”

C4. 체크인(프로그램 출석)
- 프로그램 선택 → 참가자 체크인 리스트
- 빠른 체크(스와이프/토글)
- 불참 사유(선택)
- 세션 종료 요약(참여율/특이사항)

C5. 상담/관찰 기록 타임라인
- 날짜별 기록 카드
- 기록 유형(관찰/상담/조치/연락)
- 다음 액션(리마인더 날짜 설정)

C6. 조치 계획(케어 플랜)
- 목표: 수면 규칙성, 스트레스 완화, 활동, 사회활동, 인지 루틴(선택)
- 이번 주 권장 프로그램 자동 제안 영역(수동 수정 가능)
- 진행률 체크(체크리스트)

C7. 알림 센터(이상 패턴)
- 시스템이 감지한 이벤트 리스트
- 우선순위/심각도
- 확인/보류/조치완료 플로우(Closed loop)
- “관리자에게 에스컬레이션” 버튼

C8. 보호자(자녀) 문의 처리
- 문의 리스트 + 상태(대기/처리중/완료)
- 템플릿 답변(정중, 의료 확정 표현 금지)
- 참가자 동의 범위 안내 자동 삽입

C9. 리포트 작성/요약(운영자용)
- 오늘 요약(몇 명 관리필요/주의, 특이 사건)
- 저장 후 관리자 대시보드로 공유

C10. 설정/권한 안내
- 개인정보/동의/공유 정책 요약
- 역할/권한 안내

3) 관리자 콘솔 — “치유마을 운영/식단/프로그램/성과”
------------------------------------------------
목적: 운영 통제 + 콘텐츠 관리 + 효과 분석 + 리포트

필수 화면 12개(태블릿/데스크톱 뷰 포함, 모바일 축소 버전도 가능):
A1. 관리자 메인 대시보드(통제 센터)
- 오늘 운영 KPI:
  - 참가자 수, 측정 완료율, 위험 분포(정상/주의/관리필요)
  - 프로그램 진행률/참여율
  - 이상 이벤트 수(수면 급락/스트레스 급등/낙상 등)
- “오늘의 이슈” 카드(자동 요약)
- 코디네이터 알림/보고 큐

A2. 참가자 전체 현황(필터/그룹)
- 위험도 필터
- 그룹(조/숙소/코스)별 보기
- “관리필요 탭”에서 우선 조치 대상 정리

A3. 운영 알림/이벤트 로그
- 이벤트 유형별 필터
- 조치 상태(미확인/진행/완료)
- 담당자 할당(코디네이터 지정)

A4. 프로그램 카탈로그(치유 프로그램 관리)
- 프로그램 목록(목적 태그: 수면/리듬/스트레스/인지/사회활동)
- 상세: 설명/난이도/권장 대상/정원/준비물
- CRUD(추가/복제/수정/비활성화)

A5. 프로그램 일정 캘린더
- 주간 캘린더(세션 배치)
- 정원/대기/담당 코디네이터
- 드래그 앤 드롭 편집 느낌(가능하면)
- 변경 시 공지(코디네이터/참가자)

A6. 프로그램 참여 관리
- 세션별 체크인/불참 사유
- 참여율 통계
- 참여 저조 프로그램 감지 카드
