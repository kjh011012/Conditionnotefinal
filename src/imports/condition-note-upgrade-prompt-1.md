[FIGMA MAKE UPGRADE PROMPT — 컨디션노트 v2 운영OS 강화(40+) / 코디네이터·관리자 대폭 확장]

당신은 “컨디션노트(Condition Note)”의 시니어 제품 디자이너입니다.
현재 참가자(40대 이상 성인) 앱 화면은 충분히 구축되어 있으며 유지해야 합니다.
하지만 코디네이터(캠프 진행/운영자)와 관리자(치유마을 운영 총괄) 영역이 거의 참가자 화면 복제 수준으로 빈약합니다.

목표:
- 기존 참가자 UI/톤앤매너/컴포넌트는 유지(완전 뒤엎기 금지).
- 코디네이터는 ‘개인 건강 보기’가 아니라 ‘캠프 전체 운영 + 체크리포트 + 세션진행 + 이슈관리’가 중심인 운영 OS 화면을 새로 만들어야 합니다.
- 관리자는 프로그램/식단/숙소/공지/문의/댓글을 추가·수정·삭제·배정·관리할 수 있는 관리자 콘솔을 반드시 제공해야 합니다.

필수 제약:
- iOS/Android 완전 동일 UI(동일 레이아웃).
- 모든 텍스트는 한국어.
- 의료 진단/확정 표현 금지. “지원/관리/권고/추정/참고” 톤 사용.
- 데이터: 측정기/워치 → 서버 저장 → 앱 조회(연동 화면은 ‘연동 중’이 아니라 ‘동기화 상태’ 중심).
- 이름+생년월일 일치 시 자동 다운로드 흐름 유지.
- AI(RAG + ChatGPT 3.5 Turbo)는 “요약/권고 문장 생성 UI”로만 표현(구현 설명 X).

-----------------------------------------
1) 역할별 네비게이션 구조(필수)
-----------------------------------------
- 역할 선택 화면: 참가자 / 보호자(자녀) / 코디네이터 / 관리자
- 역할 배지(헤더) 표시.
- 각 역할은 서로 다른 탭 구조를 가짐.

참가자 탭(기존 유지):
- 홈 / 기록 / 분석 / 리포트 / 설정 (예시)

코디네이터 탭(새로 설계, 반드시 참가자와 다르게):
- 캠프보드 / 체크리포트 / 세션진행 / 참가자(운영) / 이슈·문의

관리자 탭(새로 설계):
- 운영대시보드 / 프로그램 / 식단 / 숙소 / 공지 / 문의·댓글 / 리포트 / 설정(권한)

-----------------------------------------
2) 코디네이터(캠프 운영자) — 반드시 새로 만들어야 할 화면
-----------------------------------------
코디네이터는 “캠프를 운영”해야 하므로 참가자 화면 복제 금지.
코디네이터 UI는 운영 중심, 체크리포트 중심, 현장 진행 중심으로 설계.

[Coordinator Screens - 최소 12개]
C0. 캠프 선택/전환(캠프가 여러 개일 수 있음)
- 캠프 목록, 기간, 상태(진행중/예정/종료)
- 선택 시 코디네이터 운영 홈 진입

C1. 캠프보드(코디네이터 홈: 전체 상황판)
- 오늘 일정 타임라인(세션 카드: 예정/진행중/완료/지연)
- 오늘 KPI 카드:
  1) 참가자 수
  2) 측정 완료율
  3) 주의/관리필요 인원수
  4) 프로그램 참여율
- “오늘 우선 확인 대상” 리스트(관리필요/주의/측정누락/불참)
- “오늘 운영 체크리스트”(완료 체크 가능)
- 상단 알림(핵심 3개만): 이슈/문의/사고/급변

C2. 세션진행(프로그램 운영 타임라인)
- 세션 목록(시간순)
- 세션 상태 버튼: 시작/일시중지/종료
- 세션 카드에 장소/정원/담당/준비물 표시
- “체크인 시작” CTA

C3. 세션 상세 + 체크인
- 체크인 리스트(토글/스와이프)
- 불참 사유(선택)
- 세션 중 특이사항 템플릿 기록(빠른 태그 + 메모)
- 세션 종료 후 자동 요약 카드 생성(참여율/이슈/다음 조치)

C4. 체크리포트 홈(리포트 허브)
- 오늘 리포트 상태: 오전체크/저녁체크/일일요약 (초안/제출/반려/완료)
- “리포트 작성 시작” CTA
- 지난 리포트 히스토리(날짜별)
- 제출 대상(관리자) 표시

C5. 오전 체크리포트 작성(템플릿)
- 참가자 리스트(오늘 체크 필요)
- 항목(빠른 입력):
  - 수면 상태(좋음/보통/나쁨)
  - 컨디션(좋음/보통/나쁨)
  - 기분(좋음/보통/가라앉음)
  - 통증/어지러움(있음/없음)
  - 특이사항 메모
- 상단: “주의/관리필요 자동 감지” 안내 카드(참고용)
- 하단: 저장(초안), 제출 버튼

C6. 저녁 체크리포트 작성(템플릿)
- 피로/스트레스/사회활동 참여/취침 준비 상태
- 특이사항/조치 기록
- 저장/제출

C7. 일일 요약 리포트(자동 요약 + 편집)
- AI 자동 요약(1~2문단): 오늘 전반 분위기/주요 이슈
- “내일 운영 제안” 카드: 일정 조정/주의 대상
- 코디네이터 편집 가능(텍스트 편집)
- 첨부: 주요 이슈 리스트(티켓 링크)
- 제출(관리자에게 공유)

C8. 참가자(운영용) 리스트/보드
- 필터: 관리필요/주의/측정누락/프로그램 불참/체크미완료
- 정렬: 위험 우선, 최근 이슈 우선
- 참가자 카드: 숙소/조/오늘 체크 상태/최근 변화 아이콘
- 참가자 상세 진입

C9. 참가자 상세(운영용 케이스 파일) — 참가자 화면 복제가 아니라 운영 모듈 추가
- 상단: 프로필 + 오늘 체크 상태(완료/미완료)
- 핵심 지표 요약 카드(기존 컴포넌트 재사용 가능)
- 그러나 반드시 추가:
  1) “현장 기록 타임라인”(관찰/상담/조치/연락)
  2) “조치 계획/다음 확인 일정”
  3) “체크리포트 연결”(오늘 체크 입력 바로가기)
  4) “관리자에게 에스컬레이션” 버튼(이슈 생성)

C10. 이슈/인시던트(운영 이슈 관리)
- 이슈 생성(유형: 건강/안전/운영/숙소/식단/기타)
- 상태: 대기/처리중/완료
- 담당자(코디네이터/관리자) 지정
- 타임라인/첨부/내부 메모
- “리포트에 포함” 토글

C11. 문의 관리(코디네이터용)
- 참가자/보호자 문의 리스트
- 상태/우선순위/담당
- 답변 템플릿(정중 + 의료 확정 표현 금지)
- 완료 처리(Closed loop)

C12. 공지 전파/메시지(권한 범위 내)
- 코디네이터 공지(세션 변경/준비물 안내)
- 대상 선택: 참가자/보호자/전체
- 예약 발송(선택)

-----------------------------------------
3) 관리자(치유마을 운영 총괄) — 반드시 새로 만들어야 할 화면
-----------------------------------------
관리자는 아래 항목을 모두 할 수 있어야 함:
- 치유 프로그램: 추가/수정/삭제 + 일정 편성
- 식단: 추가/수정/삭제(주간 캘린더)
- 숙소 배정: 추가/수정/삭제/이동(점유 현황)
- 공지사항: 작성/수정/삭제/예약 발송
- 문의사항: 접수/담당/답변/완료
- 댓글: 모더레이션(숨김/삭제/신고 처리)
- 운영 대시보드/리포트

[Admin Screens - 최소 14개]
A1. 운영대시보드(관리자 홈)
- 오늘 KPI: 참가자 수, 측정 완료율, 주의/관리필요 분포, 프로그램 참여율
- 코디네이터 리포트 제출 현황(미제출 경고)
- 이슈/문의 처리 현황
- 오늘 공지/변경 사항

A2. 프로그램 관리(목록)
- 검색/필터(목적 태그: 수면/리듬/스트레스/인지/사회활동)
- 추가 버튼(+ 새 프로그램)
- 프로그램 카드/테이블: 이름/정원/담당/상태

A3. 프로그램 상세(수정/삭제)
- 제목, 설명, 목적 태그, 난이도, 소요시간, 장소, 준비물
- 수정/삭제/비활성화
- 관련 일정 보기

A4. 프로그램 일정 캘린더(편성)
- 주간 캘린더
- 세션 추가/이동/삭제
- 담당 코디네이터 배정
- 정원/대기 설정
- 변경 시 공지 생성 옵션

A5. 식단 관리(주간 캘린더)
- 아침/점심/저녁 카드
- 메뉴 추가/수정/삭제/복제
- 알레르기/주의 태그
- 변경 공지 자동 생성 옵션

A6. 메뉴 상세(선택)
- 원재료/주의사항/대체메뉴 메모(운영용)
- 공지 문구 템플릿

A7. 숙소 관리(숙소 구조)
- 동/층/호/침대 구조 목록
- 방 생성/수정/삭제
- 수용 인원 설정

A8. 숙소 배정(점유 그리드)
- 방 카드 그리드(점유율, 빈자리)
- 참가자 배정/이동/퇴실 처리
- 배정 변경 로그
- 과밀/빈방 경고

A9. 공지사항 관리(CRUD)
- 목록(발행/예약/임시저장)
- 작성/수정/삭제
- 대상 설정(참가자/보호자/코디네이터)
- 댓글 허용 토글, 상단 고정

A10. 문의사항 관리(티켓)
- 문의 리스트(카테고리/상태/담당/우선순위)
- 답변/내부 메모
- 담당자 지정
- 완료 처리

A11. 댓글 관리(모더레이션)
- 댓글 스트림(공지/문의/게시판)
- 신고/숨김/삭제/차단
- 모더레이션 사유 기록

A12. 사용자/권한
- 코디네이터 계정/권한
- 보호자 연결 상태
- 공유 동의 정책 안내(읽기)

A13. 리포트 센터
- 코디네이터 체크리포트 모아보기
- 주간/캠프 리포트 PDF 생성
- 운영 성과 요약(프로그램 참여율/이슈 처리율/측정 완료율)

A14. 설정(운영 정책)
- 공지 템플릿
- 문의 카테고리
- 자동 알림 규칙(측정 누락/불참 등)
- 데이터 보관 정책(안내용)

-----------------------------------------
4) UI 컴포넌트(운영자 전용으로 반드시 추가)
-----------------------------------------
- 상태 배지: 예정/진행중/완료/지연 + 정상/주의/관리필요 + 측정누락
- 타임라인 컴포넌트: 세션/이슈/기록 공용
- 체크리포트 입력 컴포넌트: 토글/3단 선택/메모/일괄완료
- 티켓 리스트 컴포넌트: 상태/담당/우선순위/마감
- 캘린더 컴포넌트: 프로그램/식단 일정
- 숙소 점유 그리드 컴포넌트: 방 카드 + 배정/이동
- “제출/반려/완료” 워크플로우 UI

-----------------------------------------
5) 상태/에러/빈화면(필수)
-----------------------------------------
- 데이터 없음(캠프 시작 전)
- 권한 없음(코디네이터가 관리자 기능 접근 시)
- 미제출 경고(체크리포트)
- 저장 중/제출 중 스테이터스
- 삭제 확인 모달(CRUD)

-----------------------------------------
6) 산출물(피그마 페이지 구성)
-----------------------------------------
Page 1: 기존 디자인 시스템 확장(운영 배지/타임라인/티켓/캘린더/숙소그리드)
Page 2: Coordinator OS (C0~C12)
Page 3: Admin Console (A1~A14)
Page 4: 보호자(자녀) (기존 정의 유지/필요 시 보강)
Page 5: User Flows (세션 진행, 체크리포트 제출, 프로그램/식단/숙소 CRUD, 문의/댓글 처리)

위 요구사항대로, 코디네이터와 관리자는 참가자 화면과 완전히 다른 ‘운영 중심 UI’로 생성하십시오.
